# MacroQuest Reverse Engineering Tooling

Tools for maintaining EQ function addresses and structure offsets across
EverQuest patches. Replaces ad-hoc manual offset hunting with a
structured, reproducible workflow.

## Patch-Day Workflow

```
EQ patches eqgame.exe
        |
        v
[1] eq_patch_diff.py old.exe new.exe --db eq_symbols.json
        |  Compares binaries, pattern-matches known symbols,
        |  reports what moved, what's missing
        v
[2] Ghidra: open new eqgame.exe
    Run mq_ghidra_bridge.py -> Import old symbols
        |  All known addresses get labels, bookmarks, comments
        |  Analyst fixes the LOST/MOVED symbols manually
        v
[3] Ghidra: analyze, confirm addresses
    Run mq_ghidra_bridge.py -> Export updated symbols
        |  Writes eq_symbols.json with new addresses
        v
[4] generate_offsets.py eq_symbols.json -f eqlib-addresses -o offsets.h
        |  Produces C++ headers compatible with eqlib
        v
[5] Update eqlib submodule with new offsets
    Build, test, commit
```

## Tools

### eq_patch_diff.py

Standalone binary comparison. No Ghidra/IDA needed.

```bash
# Compare two EQ builds
python3 eq_patch_diff.py old/eqgame.exe new/eqgame.exe --db eq_symbols.json

# Just extract version info
python3 eq_patch_diff.py --extract-version eqgame.exe
```

Reports: version changes, section layout shifts, symbol migration
(pattern-matched), and which symbols need manual RE.

### mq_ghidra_bridge.py

Ghidra script (runs inside Ghidra's Script Manager). Three modes:

- **Import**: Labels all known MQ symbols in your Ghidra project with
  bookmarks, comments, and function signatures
- **Export**: Extracts your current analysis to `eq_symbols.json`
  including auto-generated byte patterns and xrefs
- **Diff**: Compares two symbol databases (also works standalone CLI)

### generate_offsets.py

Converts symbol database to code:

```bash
# Generate eqlib-compatible C++ header
python3 generate_offsets.py eq_symbols.json -f eqlib-addresses -o offsets.h

# Generate IDA Pro import script
python3 generate_offsets.py eq_symbols.json -f ida-python -o import_mq.py

# Validate database completeness
python3 generate_offsets.py eq_symbols.json --validate
```

### eq_symbols.schema.json

JSON Schema for the symbol database. Documents every field.

## Symbol Database Format

`eq_symbols.json` is the central artifact. It tracks:

- **Functions**: Address, byte pattern, signature, category, confidence,
  xrefs, and address history across patches
- **Globals**: Variable addresses and types
- **Structs**: Layout tracking with field offsets
- **Metadata**: EQ build version, SHA256, analyst, dates

Patterns use `??` wildcards for relocatable bytes:
```
"48 8D 05 ?? ?? ?? ?? 48 89 44 24 ?? 4C 8D 0D"
```

## Adding New Symbols

1. Find the function in Ghidra
2. Label it with the eqlib naming convention (`ClassName__MethodName`)
3. Export via the bridge script
4. The bridge auto-generates the byte pattern
5. Commit the updated `eq_symbols.json`

## Critical Symbols

These MUST be present for MQ to function:

| Symbol | Purpose |
|--------|---------|
| `__MemChecker0` | CRC integrity check bypass |
| `__MemChecker1` | CRC integrity check bypass (keyed) |
| `__EncryptPad0` | CRC encryption lookup table |
| `__ModuleList` | Module enumeration (to hide MQ DLLs) |
| `__ProcessList` | Process enumeration (to hide MQ processes) |
| `__compress_block` | Compression range (patched) |
| `__decompress_block` | Decompression detour |
